[PROJECT: COMPUTER.CLI]

[RETURN_TO_GRID]

[WHAT'S_THIS?]

Computer is a personal command-line assistant that lets you run Python code, evaluate mathematical expressions, and interact with a small set of tools via natural language prompts. Think of it like a calculator and tiny coding environment combined into a single CLI utility.

Screenshot of the Computer CLI interacting with tools and running Python.

[FEATURES]

  • Evaluates math expressions (usually using sympy)
  • Execute arbitrary Python one-liners from the terminal
  • Sandboxed-ish filesystem semantics — writes limited to a sandbox directory
  • Safety checks for shell and file-write operations implemented in a simple linter
  • Supports a small set of tools (Read, Write, Bash, Confusion) for controlled interactions

[MOTIVATION]

This project began as an experiment inspired by the CodeCrafters challenge and the desire to build a system that composes small programmatic steps from natural language prompts. Instead of having the model directly answer queries, the system constructs and executes code so results are traceable and reproducible.

[INTERNALS]

The core mechanics are straightforward: the CLI accepts a natural language prompt, the agent constructs Python code (or shell commands), performs safety checks, and executes the chosen tool. The main coordination logic is in main.py and the linter lives in linter.py.

[SETUP]

This project can be installed as a CLI tool using uv. From the project root (where pyproject.toml lives), run: 

uv tool install --editable .

The project uses Groq or Ollama for inference — ensure you have GROQ_API_KEY or OLLAMA_API_KEY set in your environment if you plan to run the full system.

[USAGE]

Run directly with Python via uv: 

uv run /<PathTo>/app/main.py <your_prompt_here>

Or, if installed as a CLI tool, invoke the program with: 

computer <your_prompt_here>

[EXAMPLES]

computer what is 1+1
computer whats the integral of x^3
computer does 1=3?

Typical interaction: the system will construct a small Python snippet or shell command, run it in a sandboxed environment, and surface the computed result along with the executed tool call.
